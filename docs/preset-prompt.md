# 怎么让回复「出语音」

装了 **语音输出 (VoiceDesign)** 扩展并开启后，每条 AI 回复会被**流式朗读**。（本文以 **MiMo** 后端为例——它按角色卡里的嗓音描述现捏音色；Fish 后端改用 `reference_id`，音色部分见 README。）

你只要配：

1. **音色 base** —— 写在角色卡里
2. **一条预设 prompt** —— 让模型在正文里标情绪

---

## ① 音色 base（角色卡）

扩展从**当前角色卡**读音色（存在卡的 `data.extensions.mimo_voice_base`，随卡走）。任选：
- **让角色自己生成（推荐）**：Extensions → 语音输出 (VoiceDesign) 点「生成」——用你**当前的文本模型**、以角色口吻生成一段嗓音描述并存进卡里。
- 手动在角色卡**描述**里写：`<voice>偏中性、语速中等、带点揶揄的年轻嗓音；说话懒懒的、不紧不慢。</voice>`

固定的描述 → 声音一致（voicedesign 每次按描述现捏音色）。

## ② 预设 prompt

新建一条可开关的 system 提示词，让模型在**正文**里给要强调的地方标上表演 cue、用 `<tts>…</tts>` 包住，cue 用小米 MiMo 的音频标签，**可多个自由复合**（同一对括号内、逗号或空格分隔都行）。**通用模板**（把口吻换成你的角色）：

```
【语音输出】在你写给读者看的正文<scenario>里，给要有情绪的地方用 <tts>…</tts> 包一个表演提示（只包提示、不包台词、不加引号）；CoT/思考里别写（浪费）。提示用 MiMo 音频标签、可多个自由复合（同一对括号内，逗号或空格分隔都行）。下面每类举几个例，按情境自己挑、自己配、别照抄（音色/嗓音由角色卡定，改嗓子/方言/腔调这类别在这写）：
· 情绪/语调：委屈 无奈 释然 愤怒 惊讶 慵懒 高冷 俏皮 …
· 语速/呼吸：叹气 深呼吸 长叹一口气 停顿 语速加快 小声 …
· 状态：紧张 疲惫 撒娇 心虚 不耐烦 …
· 声音：颤抖 气声 破音 鼻音 沙哑 …
· 哭笑：轻笑 冷笑 抽泣 哽咽 …
写法示范（只示范格式，内容你按剧情定、别抄）：<tts>(紧张，深呼吸)</tts>不就个面试嘛……<tts>(小声)</tts>领带歪没歪？ 或 <tts>(委屈，哽咽)</tts>你根本没在听。
这些 <tts> 会被藏起来、只拿去指挥语音。
```

> 建议按你预设的语气重写第一句（喊角色名、用它的口吻），其余照搬。
> 上面每类只是**举例**、不是全集——完整标签见官方文档「音频标签控制」：<https://mimo.mi.com/docs/zh-CN/quick-start/usage-guide/audio/speech-synthesis-v2.5>。
> 只留**情绪/演绎**类；会改「谁在说」的（音色、御姐音等腔调、东北话等方言、唱歌）交给**角色卡的 base**，别做成逐句 cue，否则每句嗓子都在变。
> 括号半角 `()` / 全角 `（）` / `[]` 官方都支持；关键是**一律用 `<tts>` 包住**，这样正文里普通括号不会被误碰。

## ③ Regex（把 `<tts>` 藏进显示、但保留给语音）—— **扩展自动加，无需手配**

启用扩展时，它会自动往 ST 全局 Regex 里加一条隐藏 `<tts>` 的规则（`/<tts>([\s\S]*?)<\/tts>/g` → `<span class="mimo-cue">$1</span>`，Only Format Display），停用时自动移除。

（若你之前手动加过同样的规则，可以删掉，避免重复。）
