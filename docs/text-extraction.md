# 正文提取与分段 —— 适配不同回复格式指南

> 面向想让本扩展适配「自己那套回复格式」的开发者 / coding agent。
> 全部逻辑集中在 **`src/text-extract.js`** 一个文件里；改回复格式的解析，基本只动它。

TTS 只应该念「正文」（叙述 + 对白），不应该念状态栏、小剧场、思维链(CoT)、变量更新、代码块、
图片链接等「非正文」。不同角色卡 / 预设用不同写法把这些东西塞进回复里，所以「哪些该念、哪些不该念」
是**因格式而异**的——这份文档说明扩展默认怎么处理，以及你要改什么。

---

## 1. 管线总览

一条回复从「原始文本」到「可朗读的句段」经过 4 步（`src/text-extract.js`）：

```
原始 mes / 流式全量文本
  │
  ├─ regexAiOutput()      先套一遍 ST 的 AI_OUTPUT 显示正则（见 §4）
  │                        → 与 ST 最终显示/存下的 mes 对齐
  ├─ bodyTag 提取          有「正文标签」则只取 <bodyTag>…</bodyTag> 内，否则用整条
  ├─ speakClean()          去掉「看不见的块」+ 标签去壳 + markdown 标记
  └─ splitSentences()      切句 → 贪心攒到「分段上限」在句号断，各段大小一致
                           ↓
                     朗读用的句段数组
```

有 **三个入口** 调用这套管线，务必保持它们清理规则一致（原因见 §3）：

| 函数 | 用途 | 输入 |
|---|---|---|
| `getSpeakText(messageId)` | 非流式：从最终 `chat[id].mes` 提干净正文（点朗读、非流式自动播） | 消息 id |
| `streamStableText(rawFull)` | 流式：从生成中的累积全量文本提「已稳定」的干净正文 | 全量字符串 |
| `stableSegments(rawFull)` | 流式：已封口的段（供边生成边合成） | 全量字符串 |

## 2. 两个「按格式定制」的旋钮

### 2.1 正文标签 `bodyTag`（无需改代码）

设置面板「通用 → 正文标签」。留空 = 念整条可见正文；填一个标签名（如 `scenario`）=
**只念 `<scenario>…</scenario>` 里的内容**，标签外的一切（状态栏、旁注等）都不念。

这是最省事的适配方式：让你的角色卡 / 预设把要朗读的正文包在一个固定标签里，然后把标签名填进设置即可。
标签名是用户自由文本，代码里用 `reEsc()` 转义后才拼进 `RegExp`（否则 `( [ +` 等会抛异常卡死 TTS）。

### 2.2 `speakClean()` —— 「看不见的块」的清理规则（要改代码）

即使限定到正文标签内，正文里可能仍夹着不该念的东西。`speakClean(s)` 负责把它们清掉，
**这是适配新格式最常改的地方**。默认剔除（按顺序）：

```js
.replace(/<!--[\s\S]*?-->/g, " ")               // CoT（HTML 注释）
.replace(/```[\s\S]*?```|~~~[\s\S]*?~~~/g, " ")  // 代码块
.replace(/<details[\s\S]*?<\/details>/gi, " ")   // 折叠块（状态栏/小剧场/变量更新常用）
.replace(/<(script|style)[\s\S]*?<\/\1>/gi, " ")
.replace(/!\[[^\]]*\]\([^)]*\)/g, " ")           // 图片
.replace(/\[([^\]]*)\]\([^)]*\)/g, "$1")         // 链接 → 只留链接文字
.replace(/<[^>]+>/g, " ")                        // 其余标签去壳（留标签里的文字）
.replace(/[*_~`]+/g, "")                          // 强调/行内代码 标记
.replace(/\s+/g, " ").trim();
```

> ⚠️ **顺序很重要**：「看不见的块」（CoT/代码块/details）必须在「标签去壳」之前删。
> 否则 `<[^>]+>` 会先把 `<details>` 的壳去掉、只留下里面的文字，反而把状态栏内容念出来。

**示例：你的状态栏是 `[status]…[/status]` 而不是 `<details>`。** 在标签去壳那行之前加一条：

```js
.replace(/\[status\][\s\S]*?\[\/status\]/gi, " ")   // 自定义状态栏
```

**示例：你的 CoT 用 `<thinking>…</thinking>`。** ST 的显示正则(§4)通常已经把它藏了、也就不会进正文；
若没有，同样在去壳前加：

```js
.replace(/<thinking>[\s\S]*?<\/thinking>/gi, " ")
```

## 3. 铁律：三个入口的清理必须一致

`getSpeakText`（整条）、`streamStableText`（流式）、`stableSegments`（流式封口段）**必须用同一套清理**，
否则会出两类 bug：

1. **重复 / 漏读**：流式期间按「字符游标」发段、结束时 `finalizeStream`（见 `src/playback.js`）按最终
   切分补齐尾部段。如果流式清理与整条清理长度不一致，游标就错位 → 段重复或中间段丢失。
2. **念出不该念的**：如果流式清理套了某条规则、整条没套（或反之），就会念出本该隐藏的内容。

实现上三者都复用同一个 `speakClean`，且都先调 `regexAiOutput()`（§4）。**你改 `speakClean` 时，
三个入口自动一致；但如果你新增了「只在某一个入口里做的」特殊处理，务必在另外两个入口同步。**

流式还多一步 `dropUnclosedTail()`：砍掉结尾「未闭合」的构造（半条注释 / 单边代码围栏 /
未闭合 `<details>` / 半个标签 `<sce…`），避免把没写完的内容当成稳定句子提前合成。
若你新增了成对结构（如 `[status]…[/status]`）且它可能在流式中途出现，考虑在这里也补一条对应的「砍未闭合尾部」。

## 4. `regexAiOutput()` 与 ST 显示正则

`regexAiOutput(raw)`（在 `src/st.js`）会对文本套一遍 ST 的 **AI_OUTPUT 显示正则**（`regex_placement.AI_OUTPUT`）。
目的：让扩展看到的文本和 ST **最终显示/存下的 `mes`** 一致。

- 好处：用户自己配的显示正则（例如把 `<think>…</think>` 隐藏）扩展会自动跟随——被显示正则藏起来的东西，
  扩展也不会念；也保证「流式清理」和「整条清理」对齐（缓存命中、切分不错位）。
- 因此：**很多「隐藏某块」的需求，用户用 ST 自带的 Regex 扩展配一条 AI_OUTPUT 规则就能解决，不必改本扩展代码。**
- 引擎是 ST 内置扩展、异步到位；拿不到就 guarded 退回原文（不报错）。

> 历史：早先扩展会自动往 ST 全局 Regex 里塞一条隐藏 `<tts>` 的规则（`syncRegex`），现已移除。
> 「加语气」改为在合成前把标签**内联插入**到干净段上（见 `src/emote.js`），不再依赖 `<tts>` 标签，
> 所以正文里也不需要再为它保留 / 隐藏什么。

## 5. `splitSentences()` —— 分段策略

把干净正文切成句段（保留句末标点），再**贪心**把句子攒到接近「分段上限」`maxSeg`（字）、在句号处断，
使各段大小均匀。首段用独立的「首段上限」`firstSeg`（调小 = 更快出第一句）。兜底：
单句超上限 → 逗号切 → 硬切，保证每段都 ≤ 上限。

参数都在设置面板，一般无需改代码。要接入非中日标点体系（如纯英文），可调整切句正则
`[^。！？!?…\n]*[。！？!?…]+` 里的句末标点集合。

> Fish 后端注意：Fish 自身 `chunk_length` 上限就是 300、且按字数切不按整句切，`maxSeg` 建议 ≤ 300，
> 否则单段超限会被 Fish 内部再切一刀（可能切在句子中间）、听感割裂。

## 6. 下游怎么用这些段（背景，一般不用改）

- **高亮**（`src/highlight.js`）：在渲染后的 DOM 里定位当前朗读段。用「字骨架」（只留字母/数字/汉字、
  丢标点空白）比对，容忍 markdown 的印刷体替换（`...→…`、直引号→弯引号）。段文本来自「原文清理」、
  DOM 是「渲染后」文本，只有骨架比对才稳——所以段的**文字内容**不能在清理里被改写，只能整块删/去壳。
- **加语气**（`src/emote.js`）：emote 开时，对**每个干净段**送模型拿「插入指令」，把 `[情绪]` 之类标签
  **插进逐字段**里（骨架定位、只插不改字）再合成。所以清理产出的段必须是「逐字正文」——不要在清理阶段
  做同义替换 / 改写，否则骨架定位会失败、标签被丢弃。
- **合成/缓存**（`src/synth.js`）：段按 `后端+音色+段文本` 缓存。清理不稳定（同一条正文两次清出不同文本）
  会导致缓存频繁失效、重复合成。

---

## 7. 改动清单（TL;DR）

- 只想「换个正文包裹标签」：**不用改代码**，设置面板填 `bodyTag`。
- 想隐藏用户能自配的块：**优先建议用户用 ST Regex 配 AI_OUTPUT 规则**（§4），扩展自动跟随。
- 必须在扩展里剔除某种「看不见的块」：改 `speakClean()`，在「标签去壳」那行**之前**加规则（§2.2）；
  若该结构会在流式中途出现，考虑在 `dropUnclosedTail()` 也补一条（§3）。
- 改动后自检：`getSpeakText` / `streamStableText` / `stableSegments` 对同一条正文清理结果一致（§3）；
  段仍是逐字正文（§6）。
